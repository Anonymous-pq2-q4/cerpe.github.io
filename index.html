Overview

This page provides supplementary details for the real-world experiments presented in the paper. To validate the reliability of CERPE in a closed-loop setting, we designed a multi-robot formation control experiment: a leader robot is teleoperated via keyboard, while follower robots autonomously maintain a fixed relative pose to the leader using only CERPE's pose estimates.

Hardware Setup

We conduct experiments with teams of 3 and 5 Agilex Limo Pro robots in indoor environments. All onboard RGB and depth images are transmitted to a centralized laptop (as described in the paper) via TCP with image compression. The per-robot communication bandwidth is approximately 60 KB per transmission. Onboard depth sensors provide metric depth for scale recovery, replacing the Metric3Dv2 model used in simulation.

Hierarchical Control Pipeline

We employ a two-level hierarchical control architecture to bridge CERPE's perception output with robot actuation:

  • High-level (1.5–4 Hz): CERPE takes RGB images from all robots and estimates the relative poses between the leader and each follower. From these estimates, each follower's goal position is computed. The inference latency scales with the number of robots: 250 ms (2 robots), 370 ms (3 robots), and 620 ms (5 robots).
  • Low-level (15–30 Hz): Each follower interpolates the goal position from CERPE and tracks it using an MPC controller with a bicycle kinematic model. Between CERPE updates, pose is maintained via wheel odometry.

The CERPE inference latency is 250 ms (2 robots), 370 ms (3 robots), and 620 ms (5 robots), corresponding to an effective high-level control rate of approximately 1.5–4 Hz.
